# Exercise 2: Build Your Context-Engineered Prompt

## Overview

Transform a naive prompt into a context-engineered prompt using the VIBE Loop answers from Exercise 1.

## Time

12 minutes

## Instructions

1. Read the naive prompt in `before-prompt.md` — notice what is missing
2. Apply your VIBE Loop answers from Exercise 1
3. Write your context-engineered prompt in `after-prompt.md`
4. If you have access to an AI tool (Copilot, ChatGPT, Claude), test both prompts and compare the outputs

## Quality Checklist

Before you call your prompt done, verify:

- [ ] **Value** is stated explicitly (not assumed)
- [ ] **Intent** describes a real-world outcome (not just "help me write")
- [ ] **At least one boundary** is defined (what the AI must avoid)
- [ ] **Expression** matches your authentic voice (not generic corporate)
- [ ] **Guardrails** are included (no PII, no fabricated data)

## What to Look For

When you compare the naive and context-engineered outputs:

- Does the context-engineered version match your intended audience?
- Did the boundaries prevent unwanted content?
- Does the tone feel like *you* or like a generic AI response?
- Would you actually use this output, or does it need another VIBE pass?
